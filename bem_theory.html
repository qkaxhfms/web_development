BEM 방법론

Block Element Modifier

  1. Block

    재사용 가능한 기능적으로 독립적인 페이지 컴포넌트

    • 독립적인 컴포넌트 단위
    • header, search 등이 될 수 있다.

  2. Element

    자신이 속한 블럭 내에서만 의미를 가진다.

    • 블럭을 구성하는 단위
    • search의 input 등이 될 수 있다.

  3. Modifier

    • 블럭, 엘리먼트의 속성
    • search의 focused나 button의 disabled 등이 될 수 있다.

  4. __와 --로 구분한다.

    • class만을 사용한다.
    • 어떤 목적인가를 중점으로 네이밍한다.
  • 코드의 재사용성을 높히자
  • 쉽게 유지보수 하자
  • 확장 가능하게 하자
  • 클래스명만으로도 무슨 의미인지 예측 가능하도록 하자
  1. Block(전체를 감싸고 있는 블록 요소)__element(내부요소)--modifier(기능)
  2. 클래스명은 구체적이고, 명료하며 HTML안에서 읽기 쉬어야 하고, 클래스 명이 무엇을 나타내는지 분명하게 전달되어야 한다.
  3. 클래스명은 각 html태그에 오직 클래스명을 하나만 사용해야 한다.
  4. 클래스만을 활용하여 구조화하는 방법이다.
  5. 반복되는 클래스를 만들거나 똑같은 스타일을 반복해서는 안된다.
