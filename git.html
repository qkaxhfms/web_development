Git 정리하기

초기 git 설정

  1. 작업폴더에서 .gitignore 파일을 생성해서 gitHub 업로드시 제외할 폴더,파일 설정
  2. 터미널에서 git init (긱 초기화, 폴더에 .git 숨김폴더가 생성됨)
  3. git remote add origin 깃허브주소 (해당 작업포드를 깃 저장소와 연결)
  4. git config --global user.name "깃허브아이디" (깃저장소 아이디 등록)
  5. git config --global user.email "이메일주소" (깃저장소 이메일 등록)

작업시 버전별로 github 업로드

  1. '컴파일 언어 선정 후 실행 (ex gulp)'로 자동으로 컴파일 하면서 작업진행
  2. 특정 시점에서 되돌리기 포인트를 만들고 싶은 시점에 cntrl+c 로 gulp 끊어줌
  3. 그 다음 터미널에서 git add . (현재변경된 모든 내역을 git stage에 올려둠)
  4. git commi -m "커밋 메세지" (해당 변경된 내용으로 되돌리기 시점을 만들어줌)
  5. git push origin master (등록된 깃허브에 해당 해당 커밋내역들을 업로드)
  6. 다시 gulp 명령어로 watching을 하면서 작업을 다시 이어나감
  7. 다시 되돌리기 포인트 작업시 1~5번 순서까지 반복

폴더 복사해서 새로운 프로젝트를 깃허브에 등록할떄 주의점

  • 기존폴더에 .git (숨김폴더)를 무조건 삭제 (기존폴더의 작업내용이 남아서 엉킴)
  • 기존 git 폴더를 삭제한 상태에서 다시 git init해서 해당폴더의 전용의 git 세팅을 다시 시작

깃허브에 등록된 작업환경을 불러오기

  • -- git clone 깃허브 주소
    (아무것도 없는상태에서 전체 작업세팅 환경을 모두 가져올떄)
    해당 실행하는 위치가 작업 폴더를 만들고 싶은 위치에서 명렁어 실행
  • -- git pull 깃허브주소
    (이미 깃 프로젝트 환경은 세팅이 되어있는데 커밋의 버전이 다를때)
    이미 진행하고 있는 작업폴더안에서 명령어 실행

커밋지점을 자유롭게 이동하면서 확인

  1. git log (커밋 히스토리와 해당 커밋의 아이디값 확인)
  2. git checkout 커밋아이디 (해당 커밋 아이디 시점으로 되돌리기해서 확인)
  3. git checkout master (다시 최신 커밋 위치로 되돌아옴)

특정 커밋 위치에서 새로운 복사본을 생성할떄 (브랜치생성)

  1. git checkout 커밋아이디 (원하는 커밋 시점으로 이동)
  2. git branch 브렌치명 (해당 커밋시점에서 해당브랜치명으로 복사본 생성)
  3. git checkout 브렌치명 (복사한 브렌치로 이동)
